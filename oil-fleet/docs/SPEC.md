# «Флотилия» (Oil Fleet) — спека v2

**Жанр:** merge-idle. **Платформа:** Telegram Mini App, портрет. **Вид:** сверху на участок океана.

v2 = v1 (docs/spec-2 в корне репо) + механики по фидбеку: разлив нефти, танкерная логистика, вертолёт с рабочими, меню вышки.

## 1. Концепция

Сетка океана 4×4 (12 ячеек открыто, 4 выкупаются). Игрок покупает буровые вышки тира 1 и мержит две одинаковые в тир выше перетаскиванием. Вышки качают нефть в собственные склады. Нефть превращается в деньги только через **танкер**: прокликал вышки → набил танкер → отправил → танкер вернулся с выручкой.

Арт: брутальный индастриал с дофаминовой палитрой — синий океан, красно-белые платформы, чёрная нефть, сталь, оранжевые акценты. Вертолёты и танкеры анимируют сцену. Тир визуально = размер и навороченность платформы.

## 2. Кор-луп (сессия 2–4 минуты)

1. Зашёл → склады полны, возможно есть разливы → **почистил море** (тапы по пятну), восстановил добычу соседей.
2. **Харвест-логистика:** прокликал вышки → нефть стекает в танкер (сочная анимация) → отправил танкер.
3. **Купил-смержил:** монеты → вышки T1 → merge → тир выше. Назначил привезённых вертолётом рабочих.
4. Танкер вернулся (60–180 сек) → монеты упали → ещё цикл покупок → вышел.

## 3. Механики (константы — единый источник правды в `src/constants.ts`)

### 3.1 Вышка
- Тир 1–12. Добыча: `rate(tier) = 1 × 2.2^(tier−1)` барр/сек (до коэффициентов).
- Экипаж: до `tier` рабочих. Коэффициент экипажа: `0.5 + 0.5 × crew/tier`. Новая вышка — 1 рабочий.
- Склад вышки: `cap(tier) = rate(tier) × 7200` (≈2 часа полной добычи). Склад полон → добыча стоит.
- Глубина бурения (для меню вышки, чисто презентация): `depth(tier) = 40 × tier²` метров.
- Цена T1: `50 × 1.07^покупок` (счётчик покупок никогда не сбрасывается).
- **Меню вышки** (тап по вышке): глубина, скорость, склад, экипаж; кнопки «назначить рабочего», «слить в танкер», «буст ×2 на 1 ч за рекламу».

### 3.2 Merge
- Две вышки **одного тира** из любых ячеек → одна вышка `tier+1` в целевой ячейке, исходная освобождается.
- Экипаж: `min(crewA + crewB, новый tier)`. Нефть складов: `min(сумма, cap нового тира)`.
- Максимальный тир 12 — дальше merge запрещён.

### 3.3 Сетка
- 4×4. Ячейки 0–11 открыты, 12–15 покупаются: `1000 × 3^куплено`.

### 3.4 Танкер (монетизация нефти)
- Состояния: `docked` (у причала, можно грузить) → `enRoute` (отправлен) → по истечении рейса снова `docked`, выручка зачисляется **при возвращении**.
- Погрузка: тап по вышке при пришвартованном танкере → переливается `min(склад вышки, свободное место танкера)`.
- Отправка: только если загружен > 0. Выручка = `загружено × 1` монет. Рейс: 90 сек.
- Апгрейд ёмкости: `базовая 500 × 1.5^уровень`, цена `400 × 2.5^уровень`.
- Второй танкер покупается за 25 000 монет (MVP: максимум 2).

### 3.5 Разлив нефти (spill)
- Если склад вышки **полон непрерывно ≥ 600 сек** → нефть уходит в море: пятно на ячейке вышки.
- Эффект: соседние вышки (4-соседство) добывают **×0.5**. Эффекты пятен не стакаются (множитель 0.5 независимо от числа соседних пятен).
- Чистка: **5 тапов** по пятну → исчезает, добыча соседей восстанавливается.
- На вышку — максимум одно активное пятно. Пока пятно висит, повторное не появляется.

### 3.6 Вертолёт (рабочие)
- Прилетает каждые 3 часа (таймер тикает и оффлайн), привозит **2 рабочих** в общий пул (пул ≤ 6: «вертолёт не летит, если некому работать» — стимул разбирать).
- Назначение: из пула на вышку (до лимита `tier`).

### 3.7 Оффлайн
- Учитывается до **12 часов** отсутствия. Вышки качают до капов складов (с коэффициентом экипажа и активных пятен на момент выхода).
- Разливы появляются и оффлайн (та же логика 600 сек у полного склада), максимум 2 новых пятна за оффлайн — чтобы возвращение было задачей, а не разгромом.
- Танкер в рейсе возвращается оффлайн, выручка ждёт на счету при входе.

### 3.8 Дейли-стрик
- Клейм раз в UTC-сутки. Пропуск дня → сброс на 1. Награда — вышка тира по таблице `[1,1,2,2,3,3,4]` (день 1 → T1, … день 7+ → T4). Нет места на сетке → вышка в очередь подарков.

## 4. Ретеншн-механики

- Кап складов ≈2 ч → бот-пуш «Склады полны, добыча стоит».
- Разлив → бот-пуш «Нефть уходит в море!» — задача на сессию, мягкий loss-frame.
- Танкер вернулся с выручкой → пуш.
- Вертолёт привёз рабочих → пуш (не чаще 1 раза в 3 ч).
- Дейли-стрик с эскалацией наград, сгорает при пропуске.
- Лидерборд среди контактов Telegram по суммарной добыче (барр/сек) и по максимальному тиру.

## 5. Монетизация

- Rewarded video: ×2 к выручке рейса (показ при возвращении танкера — главный плейсмент), мгновенный вертолёт, буст вышки ×2 на 1 ч, мгновенный возврат танкера.
- Баннер в нижней панели меню (не на игровом поле).
- Позже: Telegram Stars (премиум-валюта: рабочие, слоты, скины флота).

## 6. MVP-скоуп

**Входит:** сетка 4×4 (12+4), тиры 1–12, merge, склады и добыча, танкер ×1 (+апгрейды, +второй), разливы и чистка, вертолёт и экипажи, меню вышки, оффлайн-прогресс, дейли-стрик, 2 rewarded-плейсмента, бот-пуши (4 триггера), лидерборд, облачное сохранение (Telegram CloudStorage + сервер позже).
**Не входит:** золотые вышки, заказы корпораций, второй участок/престиж, скины, Stars.

## 7. Технологии

TypeScript + Vite. Рендер: PixiJS (спрайты, частицы). Игровая логика — чистые функции без рендера и без побочных эффектов (`src/engine.ts`), время и rng передаются параметрами — поэтому вся экономика покрывается юнит-тестами. UI — тонкий слой поверх.

## 8. KPI теста ($300–500 Telegram Ads)

CTR креатива > 1%, CPC < $0.05, D1 > 30%, D7 > 10%, merge-действий за сессию > 5, rewarded-просмотров на DAU > 2.5.
